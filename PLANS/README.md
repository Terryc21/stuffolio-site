# Plans

Planning artifacts for stuffolio-site work. One file per planning session, dated.

## Convention

Filename: `YYYY-MM-DD-<short-slug>.md`

Each plan should:

- Reference its source (audit, conversation, decision)
- Name dependencies (other plans, strategy docs, app-repo work)
- Phase the work with effort estimates
- Include verification steps
- Document open decisions

Plans are reference artifacts, not work tickets. Active work tracking happens in the parent repo's `BETA_TO_LAUNCH_CHECKLIST.md` (when created) or in the in-progress branch.

## Cross-repo plans

When a plan spans the site repo and the Stuffolio app repo, both files reference each other. The site-facing portion lives here; the app-facing portion lives in the app repo. Plan 2 (strategy doc updates) lives in stuffolio-strategy because that is the repo it edited.
