## Phase A — Citation-rule grep automation

**Why first:** The other phases all end with a "grep to verify" step. Writing the greps once, as a script, makes Phases B-E faster AND gives the site a permanent regression guard.

**Scope:**

1. Create `stuffolio-site/scripts/site-lint.sh` with independently runnable checks:
   - `--pricing` - fails on `no subscription`, `free trial`, standalone `Pay Once. Own It.`, standalone `$9.99 unlocks everything`. Rules from `stuffolio-strategy/strategy/POSITIONING_ONE_PAGER_2026-05-07.md:100-104`.
   - `--vocab` - fails on the `stuffolio-strategy/Marketing.md:105-110` banned-word list (hoard, declutter, Marie Kondo, minimalism, insurance-grade/-approved/-ready, submit a claim, file a claim, estate planning, antique collector, collector-grade, investment-grade, AI-powered as hero claim). Plus three new rules from Plan 2's three-clause AI section: heading-position AI check, three-clause heuristic (AI sentences must contain a job-shaped phrase within ~15 words), "what it isn't" line presence on AI feature surfaces.
   - `--bylines` - fails if `Updated [Month Year]` byline drifts more than 60 days from `<time datetime="">`.
   - `--beta-debt` - fails on any remaining `BETA-PERIOD-ONLY` comment when `LAUNCH_FLIPPED=1`.
2. Each check supports an allowlist for documented intentional uses with code comments citing the rule source.
3. Output is `file:line:rule:matched-text` for direct fix targeting.
4. Top-of-script comment cites `stuffolio-strategy/strategy/SITE_LINT_PROVENANCE.md` as the rule provenance map.

**Effort:** Half a day (3-4 hours).

**Dependencies:** None.

**Verification:** Run against current `main`. Must fail on the 7+ documented hits below.
